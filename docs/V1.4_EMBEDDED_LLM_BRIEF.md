# v1.4 Brief: Embedded Low-Dimensional LLM (No API Key)

**Purpose:** Prompt for a new Cursor session to explore and implement v1.4 of the Quick Hazard Assessment app with an **embedded, low-dimensional LLM** that runs **without an API key** (local only).

---

## 1. Context

- **App:** Quick Hazard Assessment (Streamlit) — chemical hazard tool (CAS/name → PubChem, DSSTox, ToxValDB, Carcinogenic Potency Database).
- **Current v1.3:** Optional **API-based** summarization (OpenAI `gpt-4o-mini`) when `OPENAI_API_KEY` is set; rule-based CPDB summary; no local LLM.
- **Goal for v1.4:** Add a **no-API, embedded/small LLM** path for:
  - Explaining toxicity endpoints in plain language.
  - Summarizing CPDB or text excerpts.
  - Optional: SDS paste + summarize (and/or SDS lookup by CAS with links).

**Constraint:** Streamlit Community Cloud cannot run an in-process LLM (no GPU, limited RAM). So the “embedded, no API” path is **local-only**: when the user runs `streamlit run app.py` on their machine and has a local LLM runtime (e.g. Ollama), the app should use it with **no API key**.

---

## 2. Options to Compare and Implement

Evaluate and document (then implement one or more):

| Option | Description | Pros / Cons |
|--------|-------------|-------------|
| **Ollama (HTTP)** | App calls `http://localhost:11434` (or configurable `OLLAMA_HOST`) with a small model (e.g. `gemma2:2b`, `smollm:1.7b`, `qwen2:0.5b`). Use `requests` or the official `ollama` Python package. | No API key; user installs Ollama and pulls a model. Simple REST API. |
| **Ollama (Python package)** | Use `ollama` from PyPI (`pip install ollama`). Same backend as above but via Python client. | Cleaner API from code; same “Ollama must be running” requirement. |
| **Qwen2 / other via llama-cpp-python** | Load a small quantized model (e.g. Qwen2-0.5B, SmolLM-360M) **in-process** with `llama-cpp-python`. Model file can be bundled or downloaded once. | Truly embedded in the Python process; no separate Ollama server. Heavier RAM/CPU; may be slow on modest hardware. |

**Recommended exploration order:**

1. **Ollama (HTTP or Python)** — Easiest “no API key” path; works with multiple small models (Qwen, Gemma, SmolLM via Ollama). Add config: `OLLAMA_HOST`, `OLLAMA_MODEL` (e.g. `smollm:1.7b`).
2. **Optional: llama-cpp-python** — If you want a single-process, no-Ollama option, add a second backend that loads a small GGUF (e.g. SmolLM-360M or Qwen2-0.5B) when a local model path is set; document RAM/CPU requirements.

---

## 3. Where It Fits in the App

- **Config:** e.g. `config.py` or `.streamlit/secrets`: `OLLAMA_HOST` (default `http://localhost:11434`), `OLLAMA_MODEL` (default e.g. `smollm:1.7b` or `qwen2:0.5b`). Optional: `LOCAL_LLM_MODEL_PATH` for llama-cpp-python.
- **Utils:** Extend `utils/summary_utils.py` (or add `utils/local_llm.py`):
  - `is_local_llm_available()` — True if Ollama is reachable (and optionally model is pulled) or if llama-cpp model is loadable.
  - `summarize_with_local_llm(text, prompt_instruction)` — Returns summary string or None; uses Ollama or in-process model; **no API key**.
- **UI:**
  - Use local LLM for “One-sentence AI summary” (CPDB) and “Summarize excerpts with AI” (ecotoxicity text) when **no** `OPENAI_API_KEY` is set but local LLM **is** available.
  - Show a short note in sidebar or under buttons: “Using local LLM (Ollama)” or “Using embedded model” when active.
- **Docs:** README or `docs/V1.4_EMBEDDED_LLM.md`: how to install Ollama, pull a model, set `OLLAMA_HOST`/`OLLAMA_MODEL`; optional section for llama-cpp-python and a suggested GGUF model.

## 4. Technical Hints

- **Ollama REST:** `POST http://localhost:11434/api/generate` with `{"model": "smollm:1.7b", "prompt": "...", "stream": false}`. Check `GET http://localhost:11434/api/tags` to see if model is present.
- **Python `ollama`:** `import ollama; ollama.chat(model='smollm:1.7b', messages=[...])` — see [ollama-python](https://github.com/ollama/ollama-python).
- **Small models good for summarization:** `smollm:1.7b`, `qwen2:0.5b`, `gemma2:2b`, `phi3:mini` (if available in Ollama).
- Keep **OpenAI path** as-is when `OPENAI_API_KEY` is set; prefer local LLM when no key but Ollama (or local model) is available.
